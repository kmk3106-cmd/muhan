> 상태: **규칙 수집 1차 반영, 일부 미확정(아래 §7).** 코드 미작성. 사용자 최종 승인(P6) 전까지 구현/배포 없음.

## 1. 개요

밸류 리밸런싱(Value Rebalancing) = 우상향하는 **목표 밸류선 V** 에 주식 평가금을 주기적으로 맞추는 방법.
평가금이 V 위로 일정밴드 벗어나면 **매도**(현금 Pool 확보), 아래로 벗어나면 **매수**(Pool 사용). 밴드 안이면 거래 없음.
2주에 1회 판단. 종목은 TQQQ 등 레버리지 ETF 중심. **핵심은 매도** — 상단 돌파 시 주저 없이 매도해 Pool 확보.

운용 유형 3종(파라미터만 다름): **적립식 / 거치식 / 인출식**.

## 2. 핵심 용어 & 공식 (사용자 제공 — 확정)

출처: posts/009 (2022-04-15). 핵심: TQQQ 평가금을 가상 V 곡선에 맞춰 흐르게 하고 벗어나면 리밸런싱.
올인과 달리 평균 현금 13~32% 보유하면서 레버리지 수익 확보.

| 기호 | 의미 (확정) |
|---|---|
| **V** | Value. TQQQ 평가금의 기준선. 상승률 공식으로 2주마다 갱신 |
| **P (= Pool)** | **보유 현금** (P와 Pool은 동일 — 매수 재원, 매도 시 적립) |
| **밴드** | V × 0.85 ~ V × 1.15 (±15%). 벗어나면 매수/매도로 밴드 내 복귀 |
| **G (Gradient)** | V 기울기 분모. 분모 클수록 V 성장↓ → 안정적 |
| **상승률** | 다음 V에 더해지는 값 = **P/G** |
| **사이클** | 2주 1회. **V·매수표·매도표 모두 갱신** |

**핵심 공식 (확정):**
```
다음 V = 현재 V + P/G + 적립금(적립식) / − 인출금(인출식)
밴드 상단 = V × 1.15   ·   밴드 하단 = V × 0.85

평가금 > 상단  → 매도
평가금 < 하단  → 매수 (Pool 사용 한도 내)
밴드 안        → 거래 없음
```
**예시:** 매 사이클 $250 적립, 현재 V=$9,000, P(Pool)=$1,000, G=10
→ 다음 V = 9,000 + 1,000/10 + 250 = **$9,350**

### G 값 가이드 (TQQQ 3배 기준)
| G | 수익 레버리지 | 위험 레버리지 | 평균 현금보유 |
|---|---|---|---|
| /10 | 2.98배 | 2.37배 | 13.2% |
| /20 | 2.74배 | 2.18배 | ~21% |
| /30 | 2.53배 | 2.03배 | ~27% |
| /40 | 2.38배 | 1.91배 | 31.84% |

- 권장 시작: **/10**(적립식·거치식) / **/20**(인출식). 1년 단위로 /11→/12→… 점진 안정화. 금액 커지면 /30+ 권장.
- 밴드폭: **±15% 오피셜**, ±10%/±5%/±20% 커스텀 허용. **밴드폭 영향은 매우 미세 — 핵심은 G값.**

## 3. 운용 절차 (2주 사이클) — 전 유형 동일

### (1) 사이클 시작 — 새 V 계산 + 밴드 재설정
```
다음 V = 현재 V + P/G + 적립금(적립식) / − 인출금(인출식) / +0(거치식)
상단 = V × 1.15   ·   하단 = V × 0.85   (±15% 오피셜)
```

### (2) 매수/매도 판단 (평가금 = TQQQ 보유수 × 현재가)
- **평가금 > 상단(1.15V)** → **매도: 평가금이 V로 돌아올 만큼** 매도 → 받은 현금 **Pool 편입**
- **평가금 < 하단(0.85V)** → **매수: 평가금이 V에 도달할 만큼** 매수 (단 **Pool 사용 한도** 내: 적립식 75% / 거치식 50% / 인출식 25%)
- **밴드 안(0.85V~1.15V)** → 매수·매도 **없음** (거래 없는 사이클)

→ **리밸런싱 목표 = V(중심선)**. 밴드는 거래 발동 트리거일 뿐, 되돌리는 목표는 항상 V.

### (3) 예약주문 (매수표/매도표)
- 사이클 첫날 **매수점·매도점을 지정가 또는 LOC로 2주치 예약** → 사이클 중 가격 도달 시 자동 체결
- 라오어는 사이클 첫날 매수표/매도표(모델 포트폴리오 "홍길동씨")를 공유. 예약가보다 유리하게 체결돼도 사용 OK(posts/031)
- ※ **시스템 구현**: 미국 예약주문은 당일유효 → 워커가 매 거래일 매수점/매도점 지정가를 **자동 재제출**(2주 유지 효과). §6 참조.

### (4) Pool 소진 대처
- Pool 소진 시 하락장에도 매수 불가 → **대기**. 적립식은 다음 적립금으로 복구, 거치·인출식은 상승→매도로 Pool 복구될 때까지 홀드.

### (5) 매도가 핵심 (posts/021, 014)
- 밴드 상단 돌파 시 **주저 없이 매도해 Pool 확보**. **평단보다 낮아도 V 공식상 매도 신호면 매도.**

### (6) 사후 관리
- 매도가 덜 걸리거나 과매수된 경우 다음 사이클 V 갱신에서 자연 조정. 유형 전환은 수개월 단위·사전계획만(2주 단위 금지).

### 구현 관점 매수점/매도점·수량 산출식 (보유수 S, 현재가 무관·트리거가 기준)
> 평가금을 V로 되돌리는 단일 트리거 1쌍(사다리 아님 — 확인 §7-3):
- **매도점가** = 1.15V / S → 그 가격에서 평가금=1.15V. **매도수량** = S − V/(1.15V/S) = S × (1 − 1/1.15) ≈ **S × 0.1304** (평가금 V로 복귀)
- **매수점가** = 0.85V / S → **매수수량** = V/(0.85V/S) − S = S × (1/0.85 − 1) ≈ **S × 0.1765**, 단 매수금액 = 매수수량 × 매수점가 ≤ **Pool × 한도%**(초과 시 한도까지만)

## 4. 유형별 파라미터

| 항목 | 적립식 | 거치식 | 인출식 |
|---|---|---|---|
| 자금 흐름 | 매 사이클 **+적립**(월급 연동) | 없음(월급 영향 X) | 매 사이클 **−인출**(최종 목표 단계) |
| Pool 사용 한도 | 적립 후 Pool의 **75%** | Pool의 **50%** | 인출 후 Pool의 **25%** |
| 권장 G | /10 시작 | /10 시작 | /20 시작 |
| 난이도 | 쉬움 | 보통 | 어려움 |
| 라오어 중계 | VR 2~4기 | VR 0~1기 | VR 5기(2025-07~) |

- **Pool 사용 한도 = 한 사이클 매수 시 사용 가능한 현재 Pool의 비율 상한** (적립식은 적립 후·인출식은 인출 후 잔액 기준). 운용하며 점차 낮추는 방향 권장(공격/방어 선택 가능).
- 시스템·공식·매수매도는 **세 유형 완전 동일**, 파라미터(자금흐름·Pool한도·G)만 다름. 유형 전환 자유(단 수개월 단위, 2주 단위 금지).

### 유형별 상세
- **적립식**: 다음 V = V + P/G **+ 적립금**. Pool 75%. 권장 G /10(1년차 /10→/11→/12…). 월급 기반 Pool 빠른 충전, 하락장 적립금 복구. (posts/009·019·020)
- **거치식**: 다음 V = V + P/G (적립·인출 0). Pool 50%(적립식보다 보수적). 월급 무관·외부충격 강함. (posts/009·020)
- **인출식**(최종 목표): 다음 V = V + P/G **− 인출금**. Pool 25%(가장 보수적). 권장 G **/20 시작 → /30**. 원금 점차 인출, 수익분만 잔존. VR 5기(2025-07~, 원금 $2만, 팬딩 중계). (posts/009·028)

### 유형 전환 (posts/020 — 시스템 동일, 자금흐름만 차이라 전환 수월)
- **적립식→거치식**(적립 부담 시): 기존 TQQQ 개수를 거치식 모델 포트폴리오 개수에 맞춤. 남는 TQQQ 자유 처리(홀드/매도).
- **거치식→적립식**(여력 생김): 적립식 목표 N배수로 TQQQ 개수 조정(모자라면 추가 매수).
- **거치식↔인출식**(하락장 일시): 인출 부담 시 잠시 거치식처럼 운영, Pool 회복되면 인출식 복귀.
- 원칙: **수개월 단위만, 2주 단위 금지, 사전 계획 필수.** 적립식+거치식 병행이 비중 조절에 가장 유리.

## 6. 시스템 구현 관점 (trading_suite 적용)

- **미국주식 예약주문은 "당일 유효"** (미체결 장마감 후 자동취소, 휴장일만 익일 이월 — KIS 확인 완료). 따라서 "2주 한 번 걸고 방치"를 KIS 네이티브 기간예약으로는 못 함.
- → **워커가 매 거래일 매도점/매수점 지정가를 자동 재제출**해서 "2주 예약 방치" 효과를 동일하게 구현 (기존 전략의 일일 재제출 패턴 재사용).
- **V·Pool·사이클 상태**는 신규 데이터모델 필요 (트렌치 아님). 사이클 경계(2주)에 V 갱신 + 매도점/매수점 재계산.
- kis_client는 검증본 **바이트동일 복제**, 주문은 `order()` 지정가(LOC 또는 장중 §7-5). 매도 MOC는 손절 아닌 리밸런싱이라 해당 없음.

## 6.5 증권사 API 선택 — **KIS 전용** (NH/나무 부적합 확인, 2026-06)

VR도 기존 전략과 **동일하게 한국투자증권(KIS) REST API, 단일 계좌 69567573** 으로 구현한다. 다른 증권사 검토 결과:

**나무(NH투자증권) QV Open API — 미국 자동매매 부적합 (사용자 제공 SDK `Downloads/openapi.nm` 실측)**
1. **방식**: WMCA **윈도우 32비트 DLL(C/C++)** — `wmcaQuery()` 호출, 서버 `wmca.nhsec.com:8200`, bin에 `wmca.dll`·`SKComm*.dll`·`CloudNPKI.dll`(공인인증), 샘플은 VC++ 전용. **REST/Python/Linux 아님** → 우리 리눅스 NCP 서버에서 직접 불가(윈도우 PC 상시가동 + DLL 래퍼 브리지 필요).
2. **해외주식 주문 미지원**: 주문 샘플 헤더 `trio_ord.h` 해외 키워드 0건, `주문_SPEC.doc`(UTF-16 스캔) 해외·미국·나스닥·뉴욕·외화·USD **전부 0건** → 이 Open API는 **국내주식 주문 전용**. (나무 *앱*은 미국 주문 되지만 *API*는 안 됨. 시세 쪽만 '해외' 4건=해외지수류)
3. **TR 단종 예정**: KRX 대체거래소(SOR/통합시세) 개편으로 기존 TR·실시간 전문 **2026년 순차 종료** 예정 → 유지보수 리스크.

**기타 증권사**: 키움(구 OpenAPI+ 윈도우 COM, 해외 후발)·LS XingAPI·대신 CYBOS 등도 윈도우 COM/국내 중심. **미국 ETF 자동매매 REST로는 KIS가 사실상 표준**(라오어 생태계·우리 시스템 모두 KIS).

→ **결론: VR은 KIS REST로만 구현. NH/나무·타사 전환은 비권장**(윈도우 DLL·국내전용·TR단종 삼중 부적합). 단 NH가 별도 신형 REST API(미국주문 지원)를 추후 내놓으면 재검토 여지.

## 7. ⚠️ 확인 필요 (미확정 — 구현 전 사용자 답변 필수)

> 해소됨(✅): P=Pool(보유현금), G=Gradient, 상승률=P/G, Pool한도=현재Pool의 %, 밴드폭 옵션·기본±15%,
> 유형별 파라미터·전환, **리밸런싱 목표=V(중심선)**, 매수/매도 수량식, 예약주문(지정가/LOC) 방식, 적립/인출=사용자 지정 고정액, Pool 소진=대기.

남은 미확정 (P1 구현 전 확정 필요 — 대부분 실거래 등록 시 결정 가능):
1. **초기 셋업**: 신규 시작 시 **V 초기값**·**초기 주식 개수**·**Pool 시작액** 산정법. (예: 원금 중 일부로 TQQQ 매수→그 평가액=V₀, 나머지=Pool. 초기 주식:현금 비율?) — 라오어 글엔 "모델 포트폴리오 개수"로 시작하는 표현, 정확한 V₀ 산정 규칙 확인 필요.
2. **사이클 경계**: 2주 = **정확히 10거래일**인지 / 격주 특정 요일 기준인지 / 평가금·표갱신 시점(종가).
3. **매수표/매도표 단수**: 단일 트리거 1쌍(§3 구현식)으로 확정? (라오어 "표"가 다단계 사다리면 알려주세요 — 현재는 단일 1쌍으로 가정)
4. **주문 타입 기본값**: 매수점/매도점을 **LOC(종가)** vs **장중 지정가(상시)** 중 무엇으로 낼지 (둘 다 가능 — 기본 선택).
5. **시작 파라미터**: 첫 유형(적립식 추정)·G(/10)·밴드폭(±15%)·적립금액 — 실거래 등록 단계에서 확정.

- **P0** §7 미확정 사양 확정 (사용자 Q&A) — **현재 단계**
- **P1** 신규 패키지 `strategies/vr` 골격 + kis_client/kis_ws 바이트동일 복제. 신규 데이터모델(VR상태: V, Pool, G, P, 유형, 사이클번호, 보유수량/평단, 적립·인출 이력).
- **P2** VR 코어 로직(`trading_logic`): 사이클 V 갱신, 평가금 vs 밴드 판정, 매도/매수점·수량 산출, Pool 한도 적용. 단위 시뮬(백테스트 표와 대조).
- **P3** 워커: 2주 사이클 경계 처리 + 매 거래일 매도점/매수점 지정가 재제출(당일유효 대응). KIS 주문/체결 동기화.
- **P4** 부모 통합: mount·adapters(VR은 트렌치 아님 → suite_metrics 보유/사이클 VR 브랜치)·registry·schedule lane(신규)·killswitch·deploy·t_audit.
- **P4.5** 프론트 전용: 전략관리 VR 등록폼(유형/G/P/V초기/Pool한도/밴드/적립·인출금), 대시보드·포트폴리오(V선·Pool·평가금·밴드 시각화)·주문체결·성과.
- **P5** strategy-logic-validator 게이트 + 스테이징 드라이런(2주 사이클·재제출·예약 효과 실측).
- **P6** 운영 반영(**사용자 최종 승인**) — TQQQ 충돌 결정(이관 vs 타종목) + 시드·유형·파라미터 등록 후 가동.
