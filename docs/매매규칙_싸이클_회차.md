# 매매규칙 — 싸이클 / 회차(트렌치) 증감 로직 (정본)

> 본 문서는 무한매수법 V2.2·떨사오팔의 **싸이클·회차 증감 규칙의 정본(canonical spec)** 이다.
> 코드와 불일치 시 본 문서 규칙이 의도이며, 코드를 본 문서에 맞춘다.
> (2026-05-17 확정 — SOXL 부분매도 조기종료 결함 분석 결과 반영)

## 0. 용어
- **싸이클(Cycle)**: 한 종목에 대해 "매수 시작 → 보유분 전량 청산"까지의 1회전. 청산 완료 시 1 증가.
- **회차(무한매수법 buy_seq)**: 한 싸이클 안에서 N번째 분할매수. 싸이클 첫 매수 = `최초`, 이후 2,3,4…
- **트렌치(떨사오팔)**: 총액을 n등분한 매수 단위. 각 트렌치는 IDLE/BOUGHT.

## 1. 싸이클 종료 조건 (★핵심 규칙)
**싸이클은 "그 싸이클에서 매수한 물량이 전부 매도(청산)"되었을 때에만 종료한다.**

- **부분매도로는 싸이클을 종료하지 않는다.** 일부만 팔고 잔량이 남아 있으면 같은 싸이클을 계속 유지한다.
- 종료 판정은 **원장(Trade) 기준 순보유수량 = Σ(싸이클내 매수수량) − Σ(싸이클내 매도수량) ≤ 0** 을 **필수 조건**으로 한다.
- 계좌 잔고 API 의 `qty` 가 동기화 지연·체결 타이밍으로 일시 0 으로 보여도, **원장 순보유수량이 > 0 이면 종료하지 않는다.**
- (기존 안전가드 유지) 매도 체결이 1건 이상 존재 + 싸이클 내 마지막 체결이 매도 + 누적매수 > 0.

### 잘못된 동작(수정 전, 결함)
`state.qty`(계좌잔고 동기화값) 단독으로 종료 판정 → 잔고가 일시 0으로 읽히면 **부분매도 상태에서도 싸이클이 조기 종료**되고 다음 싸이클이 개시됨.
실제 사례(SOXL pid=8): C2(0424~0503) 매수 9주 중 0501에 2주만 부분매도(원장 잔량 7주)인데 C2 종료·C3 개시 → 7주가 C3에서 매도되며 C3 +777%·C2 −75% 손익 왜곡, 회차도 엉킴.

### 올바른 동작(수정 후)
원장 순보유수량 가드 추가: `net_qty(싸이클내 매수−매도 수량) > 0 → 싸이클 종료 안 함`.
→ 위 사례에서 C2는 net 7 이므로 종료되지 않고, 8주 전량 매도 시점에 1회만 종료. per-cycle 손익·회차 정합.

## 2. 회차(무한매수법 buy_seq) 증감 규칙
- 싸이클 **첫 매수 체결** → `최초`
- 같은 싸이클 내 추가 매수 체결마다 → `2, 3, 4 …` (1씩 증가)
- **싸이클이 종료(전량청산)된 경우에만** 다음 매수가 새 싸이클의 `최초` 로 리셋된다.
- 부분매도가 있어도 회차는 리셋되지 않는다(같은 싸이클이므로 계속 증가).
- 매도 체결에는 회차를 부여하지 않는다(`buy_seq = None`).

## 3. 트렌치(떨사오팔) 증감 규칙
- 총액을 `num_tranches` 등분, 각 트렌치 `amount_per_tranche` 배정.
- 전일종가 −x% 도달 → 다음 IDLE 트렌치 1칸 LOC 매수 → 해당 트렌치 BOUGHT.
- 평단가 +x% → 해당 트렌치 LOC 매도 → IDLE 복귀.
- 보유 N거래일(`loss_cut_days`) 경과 트렌치 → MOC 손절매도.
- **싸이클 종료 = 1번 트렌치가 LOC매도로 청산**되는 시점(라오어 떨사오팔 규칙). 손절(MOC)은 싸이클 미종료, 같은 싸이클 내 재매수 지속.
- 떨사오팔 역시 §1 원칙(부분 정리로 싸이클 미종료, 전량 정리 시 1회 종료)을 동일 적용.

## 4. 손익·표기 원칙
- 싸이클 손익 = (싸이클 매도합) − (싸이클 매수합). 싸이클 경계가 정확해야(=§1) per-cycle 손익이 정확.
- 경계가 어긋난 과거 싸이클의 표기는 소급 보정하지 않는다(누적·계좌 단위 KPI만 신뢰). 향후 싸이클부터 본 규칙으로 정합.
- 무한매수법 싸이클 상세의 회차 칸 = `buy_seq`(회차 N), 떨사오팔 = `tranche_num`(T#).
